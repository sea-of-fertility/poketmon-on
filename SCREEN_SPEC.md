> 각 화면의 구성 요소와 기능을 정의한 문서.
> 그래픽 없이 텍스트로만 구조를 기술한다.

|---|---|---|---|

## 1. 데스크톱 메인 뷰

앱의 핵심 화면. 투명한 윈도우 위에서 포켓몬이 자유롭게 돌아다닌다.

### 윈도우 속성
- **타입**: NSWindow (borderless, transparent)
- **배경**: 완전 투명 (clear)
- **레벨**: .floating (모든 창 위에 표시)
- **크기**: 전체 화면 (NSScreen.main 크기)
- **마우스 이벤트**: 포켓몬 영역만 클릭 가능, 나머지는 클릭 통과 (hitTest 오버라이드)

### 포켓몬 표시 영역
- **렌더링**: NSImageView 또는 CALayer 기반
- **위치**: 화면 내 자유 이동 (x, y 좌표)
- **크기**: 화면 높이의 5~8% 기본값, 사용자 슬라이더로 조절 가능
- **애니메이션**: 프레임 순환 (Walk-Anim.png에서 프레임 추출, AnimData.xml 기반 타이밍)

### 포켓몬 상태 (State Machine)

```
[Idle] ←→ [Walk] ←→ [Run]
  ↓          ↓
[Sleep]   [Reaction]
  ↑          ↑
  └── 시간경과 ──┘

[Dragged] ← 어떤 상태에서든 드래그 시 전환
```

| 상태 | 스프라이트 | 동작 설명 |
|---|---|---|
| Idle | Idle-Anim.png | 제자리에서 대기 애니메이션 반복. 일정 시간 후 Walk 또는 Sleep으로 전환 |
| Walk | Walk-Anim.png | 8방향 중 랜덤 방향으로 이동. 화면 가장자리 도달 시 방향 전환 |
| Run | Walk-Anim.png (빠른 재생) | Walk와 동일하나 이동 속도 2배, 프레임 재생 속도 1.5배 |
| Sleep | Sleep-Anim.png | 제자리에서 수면 애니메이션. 클릭 시 Idle로 복귀 |
| Reaction | Eat/Hop/Hurt-Anim.png | 특정 트리거에 의한 단발성 애니메이션. 종료 후 Idle 복귀 |
| Dragged | Idle 프레임 고정 | 마우스 드래그로 포켓몬을 원하는 위치로 이동 |

### 이동 로직
- **방향 결정**: 랜덤 목표점 생성 → dx, dy 계산 → 8방향 중 가장 가까운 방향 선택
- **속도**: 기본 2px/frame, Run 시 4px/frame (설정에서 조절 가능)
- **경계 처리**: 화면 밖으로 나가지 않음. 가장자리 도달 시 반대 방향으로 자연스럽게 전환
- **전환 타이밍**: Walk 3~10초 → Idle 2~5초 → Walk (랜덤 반복). 60초 이상 Idle 유지 시 Sleep 전환

### 인터랙션
| 입력 | 동작 |
|---|---|
| 좌클릭 | Sleep 상태면 깨우기 (→ Idle). 그 외 Reaction 트리거 |
| 드래그 | Dragged 상태로 전환, 마우스 따라 이동. 놓으면 Idle 복귀 |
| 우클릭 | 컨텍스트 메뉴(화면 3) 열기 |
| 더블클릭 | 포켓몬 선택기(화면 4) 열기 |

## 2. 메뉴바 드롭다운

macOS 상태바(메뉴바) 영역에 상주하는 아이콘과 드롭다운 메뉴.

### 메뉴바 아이콘
- **위치**: macOS 상단 메뉴바 우측 영역 (NSStatusItem)
- **아이콘**: 몬스터볼 아이콘 (16×16 또는 18×18 Template 이미지)
- **동작**: 클릭 시 드롭다운 팝오버 표시

### 드롭다운 항목

```
┌─────────────────────────────┐
│  🟢 피카츄 (Pikachu) #025   │  ← 현재 포켓몬 이름 + 번호
│  상태: 걷는 중               │  ← 현재 상태 표시
├─────────────────────────────┤
│  ▶ 포켓몬 변경...            │  → 포켓몬 선택기(화면 4) 열기
│  ⚙ 설정...                  │  → 설정 패널(화면 5) 열기
├─────────────────────────────┤
│  ⏸ 일시정지                  │  ← 토글: 모든 움직임 멈춤/재개
│  😴 재우기                   │  ← 강제 Sleep 상태 전환
│  🏃 뛰게 하기                │  ← 강제 Run 상태 전환
├─────────────────────────────┤
│  ❌ 종료                     │  ← 앱 완전 종료
└─────────────────────────────┘
```

| 항목 | 기능 | 타입 |
|---|---|---|
| 현재 포켓몬 정보 | 이름, 번호, 현재 상태를 읽기 전용 표시 | 라벨 (비활성) |
| 포켓몬 변경 | 포켓몬 선택기 윈도우를 열어 다른 포켓몬으로 교체 | 버튼 |
| 설정 | 설정 패널 윈도우 열기 | 버튼 |
| 일시정지 | 토글. 포켓몬 애니메이션과 이동을 모두 멈춤. 다시 클릭하면 재개 | 토글 버튼 |
| 재우기 | 현재 상태와 무관하게 Sleep 상태로 강제 전환 | 버튼 |
| 뛰게 하기 | Run 상태로 일시 전환 (10초 후 자동 Walk 복귀) | 버튼 |
| 종료 | NSApplication.shared.terminate 호출 | 버튼 |

## 3. 우클릭 컨텍스트 메뉴

포켓몬 스프라이트 위에서 우클릭하면 나타나는 네이티브 NSMenu.

### 메뉴 항목

```
┌───────────────────────┐
│  피카츄 #025           │  ← 헤더 (비활성)
├───────────────────────┤
│  포켓몬 변경...        │
│  설정...              │
├───────────────────────┤
│  재우기 / 깨우기       │  ← 현재 상태에 따라 텍스트 변경
│  뛰게 하기            │
├───────────────────────┤
│  종료                 │
└───────────────────────┘
```

| 항목 | 기능 |
|---|---|
| 헤더 | 현재 포켓몬 이름 + 번호 (클릭 불가) |
| 포켓몬 변경 | 화면 4 열기 |
| 설정 | 화면 5 열기 |
| 재우기/깨우기 | Sleep ↔ Idle 토글. Sleep 상태이면 "깨우기", 그 외이면 "재우기" |
| 뛰게 하기 | Run 상태로 일시 전환 |
| 종료 | 앱 종료 |

## 4. 포켓몬 선택기

사용자가 데스크톱에 표시할 포켓몬을 고르는 전용 윈도우.

### 윈도우 속성
- **타입**: NSPanel (유틸리티 윈도우)
- **크기**: 480 × 600 pt (고정 또는 리사이즈 가능)
- **레벨**: .floating (메인 뷰 위에 표시)
- **배경**: 불투명 (일반 macOS 윈도우 스타일)

### 레이아웃 구조 (위에서 아래)

```
┌──────────────────────────────────────┐
│  [검색바]                             │  ← 영역 A: 검색
│  🔍 이름 또는 번호로 검색...           │
├──────────────────────────────────────┤
│  [Gen1] [Gen2] [Gen3] [Gen4] [Gen5]  │  ← 영역 B: 세대 탭
├──────────────────────────────────────┤
│  Normal  Fire  Water  Grass  ...     │  ← 영역 C: 타입 필터 칩
│  Electric Ice  Fighting Poison ...   │
│  Ground  Flying Psychic Bug    ...   │
│  Rock    Ghost Dragon  Dark    ...   │
│  Steel   Fairy                       │
├──────────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐        │  ← 영역 D: 포켓몬 그리드
│  │ 001│ │ 002│ │ 003│ │ 004│        │
│  │icon│ │icon│ │icon│ │icon│        │
│  │name│ │name│ │name│ │name│        │
│  └────┘ └────┘ └────┘ └────┘        │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐        │
│  │ 005│ │ 006│ │ 007│ │ 008│        │
│  │    │ │    │ │    │ │    │        │
│  │    │ │    │ │    │ │    │        │
│  └────┘ └────┘ └────┘ └────┘        │
│           ... (스크롤)               │
├──────────────────────────────────────┤
│         [선택 완료]  [취소]           │  ← 영역 E: 액션 버튼
└──────────────────────────────────────┘
```

### 영역별 상세

#### A. 검색바
- **타입**: NSSearchField
- **기능**: 실시간 필터링 (타이핑과 동시에 그리드 갱신)
- **검색 대상**: 포켓몬 이름 (영문), 번호 (#025), 한글 이름 (추후)
- **동작**: 검색어 입력 시 세대 탭과 타입 필터를 무시하고 전체에서 검색. 검색어 삭제 시 원래 필터 상태 복원

#### B. 세대 탭 (Generation Tabs)
- **타입**: NSSegmentedControl (5개 세그먼트)
- **항목**: Gen 1 (151종) / Gen 2 (100종) / Gen 3 (135종) / Gen 4 (107종) / Gen 5 (140종)
- **기본값**: Gen 1 선택
- **동작**: 탭 클릭 시 해당 세대 포켓몬만 그리드에 표시. 타입 필터와 AND 조건으로 결합

#### C. 타입 필터 칩 (Type Filter Chips)
- **타입**: 가로 스크롤 가능한 NSButton 배열 (토글)
- **항목**: 18개 타입 (Normal, Fire, Water, Grass, Electric, Ice, Fighting, Poison, Ground, Flying, Psychic, Bug, Rock, Ghost, Dragon, Dark, Steel, Fairy)
- **기본값**: 전체 선택 (필터 없음)
- **동작**: 칩 클릭 시 토글 (활성/비활성). 활성화된 타입에 해당하는 포켓몬만 표시. 복수 선택 가능 (OR 조건 — 선택된 타입 중 하나라도 가진 포켓몬 표시)
- **전체 해제**: 모든 칩이 비활성이면 → 필터 없음과 동일 (전체 표시)

#### D. 포켓몬 그리드
- **타입**: NSCollectionView (그리드 레이아웃)
- **셀 크기**: 80 × 90 pt (아이콘 + 이름)
- **셀 구성**:
  - 상단: Idle-Anim.png 첫 프레임 (또는 Walk-Anim.png 첫 프레임) 축소 표시
  - 하단: 포켓몬 이름 (영문, 작은 글씨)
  - 번호: 셀 좌상단에 작게 표시 (#025)
- **선택 표시**: 클릭 시 파란색 테두리 하이라이트
- **현재 사용 중 표시**: 현재 데스크톱에 있는 포켓몬은 체크마크 또는 녹색 테두리
- **스프라이트 없는 포켓몬**: 누락 16종은 회색 처리 + "N/A" 표시, 선택 불가
- **스크롤**: 세로 스크롤

#### E. 액션 버튼
- **선택 완료**: 선택한 포켓몬으로 데스크톱 펫을 교체. 윈도우 닫기
- **취소**: 변경 없이 윈도우 닫기

### 필터 조합 로직
```
최종 결과 = 검색어 필터 (있으면 우선)
           OR (세대 탭 AND 타입 필터)

예시:
- Gen 1 탭 + Fire 칩 → Charmander, Charmeleon, Charizard, Vulpix, Ninetales, ...
- Gen 1 탭 + Fire + Flying 칩 → 위 결과 + Pidgey, Pidgeotto, ... (Fire 또는 Flying)
- 검색 "pika" → Pikachu, Pikachu 관련 포켓몬 (세대/타입 무관)
```

## 5. 설정 패널

포켓몬 동작과 표시를 조절하는 설정 윈도우.

### 윈도우 속성
- **타입**: NSPanel
- **크기**: 400 × 500 pt
- **레벨**: .floating

### 설정 항목

```
┌──────────────────────────────────────┐
│  ⚙ 설정                              │
├──────────────────────────────────────┤
│                                      │
│  표시                                │
│  ──────────────────────────────      │
│  크기    [━━━━━●━━━] 120%            │
│          작게          크게           │
│                                      │
│  투명도  [━━━━━━━●━] 90%             │
│          투명          불투명         │
│                                      │
│  표시 위치 [항상 위 (모든 창 위)  ▼]  │
│                                      │
│  행동                                │
│  ──────────────────────────────      │
│  이동 속도  [━━●━━━━━━] 느림          │
│            느림         빠름          │
│                                      │
│  활동 빈도  [━━━━●━━━━] 보통          │
│            조용함       활발함        │
│                                      │
│  수면까지   [━━━━━━●━━] 3분           │
│            1분          10분          │
│                                      │
│  시스템                              │
│  ──────────────────────────────      │
│  ☐ 로그인 시 자동 실행               │
│                                      │
├──────────────────────────────────────┤
│    [기본값 복원]           [닫기]     │
└──────────────────────────────────────┘
```

### 항목별 상세

#### 표시 섹션

| 항목 | 타입 | 범위 | 기본값 | 설명 |
|---|---|---|---|---|
| 크기 | NSSlider | 50% ~ 200% | 100% (화면 높이 5~8% 자동 계산) | 포켓몬 스프라이트의 렌더링 크기. 모니터 해상도 기반 기본값에 배율 적용 |
| 투명도 | NSSlider | 30% ~ 100% | 100% | 포켓몬 스프라이트의 불투명도. 낮출수록 반투명 |
| 표시 위치 | 드롭다운 | 항상 위 / 일반 / 바탕화면만 | 항상 위 (모든 창 위) | 포켓몬 윈도우의 레벨. "항상 위"=.floating, "일반"=.normal, "바탕화면만"=.desktop |

#### 행동 섹션

| 항목 | 타입 | 범위 | 기본값 | 설명 |
|---|---|---|---|---|
| 이동 속도 | NSSlider | 1 ~ 5 단계 | 3 (보통) | Walk/Run 시 프레임당 이동 픽셀 수 조절 |
| 활동 빈도 | NSSlider | 1 ~ 5 단계 | 3 (보통) | Walk ↔ Idle 전환 주기. 높을수록 자주 움직임 |
| 수면까지 | NSSlider | 1분 ~ 10분 | 3분 | Idle 상태 유지 시 Sleep으로 자동 전환되는 시간 |

#### 시스템 섹션

| 항목 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| 로그인 시 자동 실행 | NSButton (체크박스) | OFF | macOS 로그인 항목에 등록/해제 (SMAppService 또는 LaunchAgent) |

#### 액션 버튼

| 버튼 | 기능 |
|---|---|
| 기본값 복원 | 모든 설정을 기본값으로 리셋 |
| 닫기 | 설정 패널 닫기. 변경사항은 실시간 반영 (별도 저장 버튼 불필요) |

### 데이터 저장
- **저장소**: UserDefaults
- **키 프리픽스**: `com.poketmon.settings.`
- **저장 시점**: 슬라이더/체크박스 값 변경 즉시 저장
- **불러오기**: 앱 실행 시 자동 로드, 값 없으면 기본값 사용

## 화면 전환 흐름

```
앱 실행
  │
  ├──→ [1. 데스크톱 메인 뷰] (항상 표시)
  │         │
  │         ├── 우클릭 ──→ [3. 컨텍스트 메뉴]
  │         │                    ├── 포켓몬 변경 → [4. 포켓몬 선택기]
  │         │                    ├── 설정 → [5. 설정 패널]
  │         │                    └── 종료 → 앱 종료
  │         │
  │         └── 더블클릭 ──→ [4. 포켓몬 선택기]
  │
  └──→ [2. 메뉴바 드롭다운] (항상 접근 가능)
               ├── 포켓몬 변경 → [4. 포켓몬 선택기]
               ├── 설정 → [5. 설정 패널]
               ├── 일시정지/재우기/뛰게하기 → 데스크톱 뷰 상태 변경
               └── 종료 → 앱 종료
```

### 주요 흐름별 시나리오

**시나리오 1: 첫 실행**
1. 앱 실행 → 데스크톱 메인 뷰 표시
2. 기본 포켓몬 (피카츄 #025)이 화면 중앙에 Idle 상태로 등장
3. 2초 후 Walk 상태로 전환, 랜덤 방향으로 이동 시작

**시나리오 2: 포켓몬 변경**
1. 메뉴바 아이콘 클릭 → 드롭다운 → "포켓몬 변경" 클릭
2. 포켓몬 선택기 윈도우 열림
3. Gen 탭 선택 → 타입 필터 → 원하는 포켓몬 클릭 → "선택 완료"
4. 데스크톱의 포켓몬이 새 포켓몬으로 교체 (현재 위치에서 페이드 전환)
5. 선택기 윈도우 닫힘

**시나리오 3: 크기 조절**
1. 우클릭 → "설정" → 설정 패널 열림
2. 크기 슬라이더를 150%로 드래그
3. 데스크톱의 포켓몬이 실시간으로 커짐 (설정 패널 닫지 않아도 즉시 반영)
4. "닫기" 클릭

**시나리오 4: 상호작용**
1. 포켓몬이 Walk 중 → 사용자가 클릭 → Reaction (Hop 애니메이션 1회 재생)
2. Reaction 종료 → Idle 복귀
3. 사용자가 포켓몬을 드래그 → Dragged 상태, 마우스 따라 이동
4. 마우스 놓기 → Idle → 다시 Walk 시작

## 기술 노트

### 멀티 모니터 지원
- 기본: 주 모니터(NSScreen.main)에서만 이동
- 향후 확장: 모니터 간 이동 지원 가능 (NSScreen.screens 배열 활용)

### 접근성
- 메뉴바 드롭다운은 키보드 접근 가능 (macOS 기본 NSMenu 동작)
- VoiceOver: 포켓몬 이름과 상태를 접근성 레이블로 제공

### 성능
- 프레임 업데이트: CADisplayLink 또는 NSTimer (60fps 불필요, 15~30fps 충분)
- 스프라이트 캐싱: 선택된 포켓몬의 모든 프레임을 앱 시작 시 메모리에 로드
- 메모리: 포켓몬 1종 기준 Walk+Idle+Sleep 프레임 전체 ≈ 1~3MB

### 저장 데이터
- 마지막 선택 포켓몬 ID
- 설정 패널의 모든 값
- 마지막 위치 (앱 재실행 시 같은 위치에서 시작)
